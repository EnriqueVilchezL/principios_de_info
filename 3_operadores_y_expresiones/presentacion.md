---

**Tabla de Operadores Aritm√©ticos:**

## ‚öñÔ∏è Operadores Relacionales (Comparaci√≥n)

---

Nos permiten **comparar** dos valores.
El resultado de una comparaci√≥n siempre es un valor **booleano**: `True` o `False`. ‚úÖ‚ùå

**Tabla de Operadores Relacionales:**

| Operador | Nombre                   | Ejemplo       | Resultado |

| Operador | Nombre             | Ejemplo       | Resultado |

---

**Ejemplo Pr√°ctico:**

## üí° Operadores L√≥gicos

---

Nos permiten **combinar** o **modificar** expresiones booleanas.
Son esenciales para tomar decisiones complejas en tu c√≥digo. üö¶

---

**Los tres operadores l√≥gicos principales:**
*   `and` (Y l√≥gico)
*   `or` (O l√≥gico)
*   `not` (Negaci√≥n l√≥gica)

---

**`and` (Y l√≥gico):**
Retorna `True` si **AMBAS** condiciones son `True`.
Si una es `False`, el resultado es `False`.

**Tabla de `and`:**

| Condici√≥n 1 | Condici√≥n 2 | Resultado |
|-------------|-------------|-----------|
| `True`      | `True`      | `True`    |
| `True`      | `False`     | `False`   |
| `False`     | `True`      | `False`   |
| `False`     | `False`     | `False`   |

---

**`or` (O l√≥gico):**
Retorna `True` si **AL MENOS UNA** de las condiciones es `True`.
Solo es `False` si **AMBAS** son `False`.

**Tabla de `or`:**

| Condici√≥n 1 | Condici√≥n 2 | Resultado |
|-------------|-------------|-----------|
| `True`      | `True`      | `True`    |
| `True`      | `False`     | `True`    |
| `False`     | `True`      | `True`    |
| `False`     | `False`     | `False`   |

---

**`not` (Negaci√≥n l√≥gica):**
Invierte el valor booleano. `True` se vuelve `False` y viceversa.

**Tabla de `not`:**

| Condici√≥n | Resultado |
|-----------|-----------|
| `True`    | `False`   |
| `False`   | `True`    |

---

**Ejemplo Pr√°ctico:**
```python
es_dia_soleado = True
tengo_sombrilla = False
voy_a_playa = True

# ¬øPuedo ir a la playa SI hace sol Y tengo sombrilla?
print(f"¬øPlaya con sol y sombrilla? {es_dia_soleado and tengo_sombrilla}") # False

# ¬øPuedo ir al cine SI hace sol O tengo sombrilla?
print(f"¬øCine con sol o sombrilla? {es_dia_soleado or tengo_sombrilla}") # True

# ¬øNO voy a la playa?
print(f"¬øNo voy a la playa? {not voy_a_playa}") # False
```

## üÜî Operadores de Identidad (`is`, `is not`)

---

Nos permiten verificar si dos variables se refieren al **mismo objeto en memoria**.
Es diferente de `==` que compara solo los *valores*.
Piensa: "¬øSon estas dos llaves exactamente la MISMA llave?" üîëüîë

---

**Operadores de Identidad:**
*   `is`: Retorna `True` si ambos operandos son el mismo objeto.
*   `is not`: Retorna `True` si ambos operandos **NO** son el mismo objeto.

---

**Diferencia clave: `is` vs `==`**
*   `==` compara **valores**.
*   `is` compara **identidad** (si es el mismo objeto).

```python
lista1 = [1, 2, 3]
lista2 = [1, 2, 3] # Nueva lista, mismo contenido
lista3 = lista1    # 'lista3' ahora apunta al MISMO objeto que 'lista1'

print(f"lista1 == lista2: {lista1 == lista2}") # True (mismo valor)
print(f"lista1 is lista2: {lista1 is lista2}") # False (objetos diferentes en memoria)

print(f"lista1 == lista3: {lista1 == lista3}") # True (mismo valor)
print(f"lista1 is lista3: {lista1 is lista3}") # True (es el mismo objeto)
```

---

**Ejemplo con n√∫meros y cadenas (inmutables):**
Para tipos inmutables (n√∫meros, cadenas), Python a veces optimiza y usa el mismo objeto para valores id√©nticos.
Pero no siempre debes confiar en `is` para comparar valores de inmutables. Usa `==`.

```python
a = 10
b = 10
print(f"a is b: {a is b}") # True (Python optimiza enteros peque√±os)

s1 = "Hola"
s2 = "Hola"
print(f"s1 is s2: {s1 is s2}") # True (Python optimiza cadenas cortas)

s3 = "Esta es una cadena muy larga que Python no siempre optimiza igual"
s4 = "Esta es una cadena muy larga que Python no siempre optimiza igual"
print(f"s3 is s4: {s3 is s4}") # Puede ser True o False, depende de la implementaci√≥n de Python
```

# Evaluaci√≥n de Expresiones üß†

---

Cuando tienes una expresi√≥n con varios operadores, ¬øen qu√© orden se ejecutan?
¬°Aqu√≠ entran la **precedencia** y la **asociatividad**!

---

## Precedencia de Operadores üèÜ

---

La **precedencia** determina el orden en que se eval√∫an los operadores.
Es como el "orden de las operaciones" en matem√°ticas (PEMDAS/BODMAS).

---

**Orden general (de mayor a menor precedencia):**
1.  **Par√©ntesis `()`**: ¬°Lo que est√° dentro se eval√∫a primero! üöÄ
2.  **Exponenciaci√≥n `**`**
3.  **Multiplicaci√≥n `*`, Divisi√≥n `/`, Divisi√≥n Entera `//`, M√≥dulo `%`**
4.  **Suma `+`, Resta `-`**
5.  **Comparaci√≥n `==`, `!=`, `>`, `<`, `>=`, `<=`**
6.  **Identidad `is`, `is not`**
7.  **Membres√≠a `in`, `not in`**
8.  **Negaci√≥n l√≥gica `not`**
9.  **AND l√≥gico `and`**
10. **OR l√≥gico `or`**

---

**Ejemplo de Precedencia:**
```python
resultado = 5 + 3 * 2 # ¬øEs 16 o 11?
# Multiplicaci√≥n (*) tiene mayor precedencia que Suma (+)
# 1. 3 * 2 = 6
# 2. 5 + 6 = 11
resultado = 11
print(resultado)
```

---

**Usando Par√©ntesis para Forzar el Orden:**
```python
resultado_forzado = (5 + 3) * 2
# Par√©ntesis se eval√∫a primero
# 1. (5 + 3) = 8
# 2. 8 * 2 = 16
resultado_forzado = 16
print(resultado_forzado)
```
¬°Los par√©ntesis son tus amigos para la claridad! üòâ

---

## Asociatividad de Operadores ‚ÜîÔ∏è

---

La **asociatividad** determina c√≥mo se eval√∫an los operadores de la **misma precedencia**.
¬øDe izquierda a derecha o de derecha a izquierda?

---

**Mayor√≠a de operadores en Python:**
Son **asociativos de izquierda a derecha**.
Ejemplo: `10 - 5 - 2`
` (10 - 5) - 2 ` -> ` 5 - 2 ` -> ` 3 `

---

**Excepci√≥n importante: Exponenciaci√≥n `**`**
Es **asociativa de derecha a izquierda**.
Ejemplo: `2 ** 3 ** 2`
` 2 ** (3 ** 2) ` -> ` 2 ** 9 ` -> ` 512 `

---

**Ejemplo de Asociatividad:**
```python
print(10 - 5 - 2) # (10 - 5) - 2 = 5 - 2 = 3
print(2 ** 3 ** 2) # 2 ** (3 ** 2) = 2 ** 9 = 512
```
